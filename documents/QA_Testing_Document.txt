Supplier Management System
QA and Testing Document

1. Introduction

This QA and Testing Document is prepared for the Supplier Management System. The purpose of this document is to check whether all major features of the system are working correctly or not.

2. Testing Objective

1. Verify that the system works according to requirements.
2. Check all CRUD operations.
3. Check form validation.
4. Check calculation logic.
5. Check login and logout functionality.
6. Check responsive design.
7. Identify and fix bugs before submission.
8. Make sure dummy data is used only.

3. Test Environment

Project Name: Supplier Management System
Frontend: React.js
Backend: Node.js and Express.js
Database: MongoDB
Browser Used: Google Chrome
Testing Type: Manual Testing

4. Test Cases

Steps: Open website URL and check Home, About, Features, Contact, and Login pages.
Expected Result: All public pages should open properly.
Status: Passed
